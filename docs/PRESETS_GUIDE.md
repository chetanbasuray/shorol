# Contributor Guide for Adding Presets Safely

## Introduction
This guide aims to provide contributors with the necessary steps and precautions to add presets safely to the project.

## Steps to Add a Preset
1. **Understand the Existing Code:** Before adding a new preset, review the existing presets and understand their structure and functionality.
2. **Create a New Branch:** Always create a new feature branch based on the latest `feat/1.5.x-milestone` branch for your changes.
3. **Add Your Preset:** Follow the designated format for adding your preset. Ensure you keep the naming conventions consistent.
4. **Document Your Changes:** Provide a brief description of your preset and any necessary details in this guide.
5. **Test Your Changes:** Thoroughly test your preset to ensure it works as expected without causing issues in the existing setup.
6. **Create a Pull Request:** Once you’re satisfied with your changes, submit a pull request for review.

## Best Practices
- **Code Consistency:** Ensure your code follows the project's coding standards.
- **Version Control:** Keep track of versioning for presets, and clearly indicate any major changes made.
- **Backup Data:** Make sure to backup any data or settings that could be affected before making major changes.

## Conclusion
By following these guidelines, you contribute to maintaining the integrity and functionality of the project while allowing new presets to be added safely.
